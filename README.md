# MenuPlanner 🍽️

**MenuPlanner** — мобільний додаток для Android, розроблений для управління тижневим раціоном та організації персональної колекції рецептів. Проєкт пропонує реактивний та зрозумілий інтерфейс для відстеження калорійності, планування приготування їжі та збереження власних страв.

## 🚀 Технологічний стек

Проєкт реалізовано з використанням наступного набору інструментів native-розробки для Android:
* **Kotlin:** Основна мова програмування.
* **Jetpack Compose:** Сучасний декларативний інструментарій для побудови інтерфейсу.
* **Material Design 3 (M3):** Стилізація, типографіка та стандартні UI-компоненти (Scaffold, Cards, Snackbars, FAB тощо).
* **Jetpack Navigation Compose:** Управління маршрутизацією та передачею аргументів між екранами.

## 🗂️ Моделі даних

Основна логіка додатка базується на двох ключових моделях, які керуються реактивно в пам'яті (`DummyData.kt`):

### 1. `Recipe` (Рецепт (Страва))
Представляє конкретну страву, яку можна додати до плану харчування.
* `id` (UUID): Унікальний ідентифікатор.
* `title` (String): Назва страви.
* `description` (String): Текстове поле для детального опису інструкцій приготування або інгредієнтів.
* `calories` (Int): Енергетична цінність в калоріях.
* `prepTimeMinutes` (Int): Час приготування в хвилинах.
* `isVegetarian` (Boolean): Ознака вегетаріанської страви.
* `dateAdded` (Date): Дата створення рецепта.

### 2. `MealPlan` (План (Меню) харчування)
Представляє розклад страв (план харчування, або ж умовне меню) на конкретний день тижня.
* `id` (UUID): Унікальний ідентифікатор.
* `dayOfWeek` (String): День тижня (Понеділок — Неділя).
* `breakfast`, `lunch`, `dinner` (Recipe): Рецепти, призначені на сніданок, обід та вечерю відповідно.
* `isCooked` (Boolean): Статус, що вказує, чи було реалізовано (приготовано) план на цей день.

## 📱 Екрани та навігація

Додаток використовує **Bottom Navigation Bar** з трьома основними вкладками та вкладеним (nested) графом навігації для детальних переглядів і форм редагування.

### 📅 Вкладка 1: Плани харчування
* **`MealPlanListScreen`:** Огляд тижня (7 днів). Представлено у вигляді списку. Відображає підсумок обраних страв та статус виконання плану.
* **`MealPlanDetailScreen`:** Детальна інформація про меню на конкретний день тижня. Відображає назви страв що входять до меню, загальну калорійність меню, чи є меню вегетаріанським та дає можливість відмітити меню як "Приготоване". Містить швидкі посилання для перегляду конкретних рецептів.
* **`MealPlanUpdateScreen`:** Форма для зміни меню дня. Використовує випадаючі списки (`ExposedDropdownMenuBox`) для вибору рецептів з візуальними підказками (калорії, іконки для вегетаріанських страв).

### 📖 Вкладка 2: Рецепти
* **`RecipeListScreen`:** Список усіх доступних рецептів. Картки візуально відрізняються залежно від типу страви (зелений відтінок для вегетаріанських, червоний - для всіх інших). Наявна **"плаваюча" кнопка (`floatingActionButton`)** для додавання нового рецепту (страви). Підтримує **Swipe-to-delete** для видалення.
* **`RecipeDetailScreen`:** Деталі рецепта та інтерактивний **блок опису**. Користувач може редагувати інструкції прямо на цьому екрані.
* **`RecipeUpdateScreen`:** Спеціальний екран для редагування загальних параметрів (назва, калорії, час приготування, тип страви).
* **`RecipeCreateScreen`:** Форма для додавання нового рецепту.

### 👤 Вкладка 3: Профіль
* **`ProfileScreen`:** Екран-заглушка для майбутнього розширення функціоналу.

## ✨ Основні та інші реалізовані функції

* **Реактивний інтерфейс:** Завдяки використанню (`mutableStateOf`), будь-які зміни в даних (наприклад, редагування назви рецепта) миттєво відображаються у всіх частинах додатка.
* **Перехоплювач незбережених змін:** Якщо користувач редагував опис (`description`) рецепта і намагається повернутись на попередній екран (через системну кнопку "Назад" або кнопку '←' в тулбарі), додаток перехоплює дію та виводить діалогове вікно з попередженням.
* **Система сповіщень (Snackbars):** Додаток інформує про успішне оновлення даних або створення нових елементів за допомогою коротких спливаючих повідомлень, використовуючи `SavedStateHandle` для передачі статусу між екранами.
* **Візуальна диференціація:** Інтерфейс динамічно підлаштовується під контент, виділяючи вегетаріанські опції кольором та іконками, що полегшує процес планування.
